# Exploring-Patient-Visit-Data-in-Excel-Healthcare-Informatics-Lab
Project Overview

This project focuses on the classification and analysis of patient visit data across various healthcare data sources. The goal was to demonstrate how data flows through different systems—such as Electronic Health Records (EHR), Claims, Registries, and Clinical Trials—to create a holistic view of a patient’s journey.
Learning Objectives

    Differentiate between primary healthcare data sources.

    Classify clinical and administrative data fields.

    Utilize Excel Pivot Tables to aggregate and summarize healthcare datasets.

The Dataset

The dataset simulates a patient journey for "John Smith," who visited a clinic for a persistent cough. Data elements included:

    Clinical Data: Diagnoses, medications, and lab results.

    Administrative Data: Billing codes and insurance information.

    Research/Longitudinal Data: Chronic condition tracking and clinical trial participation.

Key Tasks Performed

    Data Categorization: Manually audited records to assign them to their likely source (EHR vs. Claims vs. Registry).

    Quantitative Analysis: Used sorting and counting methods to verify data distribution.

    Data Summarization: Built a Pivot Table to visualize the count of records per data source.

    Strategic Matching: Identified which data source is best suited for specific analytical goals (e.g., using Claims for cost analysis and EHR for clinical history).

Tools Used

    Microsoft Excel (Data Validation, Sorting, Pivot Tables)

    Markdown (for documentation)
